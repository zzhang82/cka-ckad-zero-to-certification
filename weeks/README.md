# Weekly Orchestration

Week folders are thin execution manifests. They define the target, ordered references, timebox, evidence, and acceptance gate for one week.

They do not contain duplicate copies of canonical lessons, labs, scenarios, or solutions.

The learner always opens `weeks/week-XX/START_HERE.md` first.

Each week must contain:

- `START_HERE.md` for the actionable dashboard;
- `README.md` for the human execution plan;
- `week.json` for stable IDs, resource references, deliverables, and gate metadata.

Use [the week template](../templates/WEEK_TEMPLATE.md) and [START_HERE template](../templates/WEEK_START_HERE_TEMPLATE.md) when adding a new week.
