# Week 11 Resources

Do not start another broad course. Select the smallest official source that addresses the measured failure.

## Always use

1. [Current CNCF curricula](https://github.com/cncf/curriculum) — classify the miss against the current objective rather than a community chapter name.
2. [`kubectl` quick reference (v1.35)](https://v1-35.docs.kubernetes.io/docs/reference/kubectl/quick-reference/) — deliberate speed drills for slow command retrieval.
3. [Deprecated API migration guide (v1.35)](https://v1-35.docs.kubernetes.io/docs/reference/using-api/deprecation-guide/) — API-version and migration gaps.
4. [Troubleshooting applications (v1.35)](https://v1-35.docs.kubernetes.io/docs/tasks/debug/debug-application/) — repeat Pod, Service, init-container, logs, and event failures.

Then select exactly one relevant official page from Weeks 1–9. Stop after two changed, independent scenarios succeed without hints.

Optional community repositories may supply exercise ideas only after license, commit, blueprint, and v1.35 checks; their old weight tables are never authoritative.
