# Week 9 Resources

## Read before practice

1. [Configure liveness, readiness, and startup probes](https://v1-35.docs.kubernetes.io/docs/tasks/configure-pod-container/configure-liveness-readiness-startup-probes/) — choose and debug the correct probe.
2. [Logging architecture (v1.35)](https://v1-35.docs.kubernetes.io/docs/concepts/cluster-administration/logging/) — current, previous, and multi-container logs.
3. [Troubleshooting applications](https://v1-35.docs.kubernetes.io/docs/tasks/debug/debug-application/) — primary broken-state workflow.
4. [Services (v1.35)](https://v1-35.docs.kubernetes.io/docs/concepts/services-networking/service/) — selector, targetPort, and EndpointSlice diagnosis.
5. [NetworkPolicy (v1.35)](https://v1-35.docs.kubernetes.io/docs/concepts/services-networking/network-policies/) — isolation and additive semantics; labs require an enforcing CNI.
6. [Ingress (v1.35)](https://v1-35.docs.kubernetes.io/docs/concepts/services-networking/ingress/) — current CKAD blueprint surface.

Gateway API is not a CKAD core resource in this baseline, and the Linux Foundation allowlist currently identifies its documentation for CKA only. Recheck the [live allowed-resource page](https://docs.linuxfoundation.org/tc-docs/certification/certification-resources-allowed) before the exam.
