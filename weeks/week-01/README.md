# Week 1 — API Objects and Command Fluency

> Preview: resources and acceptance intent are published; the original labs and grader are not yet runnable.

**Goal:** turn rusty Kubernetes recognition into fast, verified object work with Pods, Deployments, Services, namespaces, labels, and YAML.

**Before starting:** pass Week 0 or carry an explicit remediation plan. Read [this week's resources](RESOURCES.md), then map the matching early LFS258 modules without copying course content.

## Planned work

1. Retrieve the component, object, spec/status, and controller models from memory.
2. Generate and edit manifests with `kubectl`, then verify the API result rather than the input file.
3. Complete a guided workload-and-Service lab.
4. Repair a broken Pod and selector path from events and object state.
5. Complete a timed object-creation check.

## Planned gate

- Create and explain the required objects without a solution.
- Use explicit context and namespace targeting.
- Diagnose one Pod and one Service failure from decisive evidence.
- Record attempts in `weeks/week-01/EVIDENCE.md` under your configured learner directory.
