# CKA → CKAD 12-Week Target Map

Version: 0.1
Last updated: 2026-07-15

The map assumes roughly 10–12 focused hours per week. Course module titles will be added after the purchased LFS258/LFS259 tables of contents are inventoried.

| Week | Short target | Primary spine | Exit evidence |
|---|---|---|---|
| 0 | Prove the environment and establish the learner baseline | Official exam instructions, tool docs, current blueprints | Two cluster rebuilds, diagnostic, remediation map |
| 1 | Reactivate API-object, YAML, namespace, and command fluency | Early LFS258; Kubernetes objects, Pods, controllers | Guided workload lab, broken Pod drill, timed object task |
| 2 | Explain architecture, access, resources, and scheduling | LFS258; components, RBAC, ServiceAccounts, scheduling | Architecture retrieval, RBAC/scheduling scenarios, timed mixed check |
| 3 | Build and maintain clusters safely | LFS258; kubeadm, lifecycle, upgrades, etcd, Helm/Kustomize | Disposable-node prototype, upgrade/recovery drill |
| 4 | Master networking and storage | LFS258; Services, DNS, NetworkPolicy, Gateway/Ingress, PV/PVC/StorageClass | Network and storage failure drills, timed domain set |
| 5 | Troubleshoot across application, node, and control-plane layers | Finish LFS258; official debug tasks | Mixed debugging set, current CKA coverage report |
| 6 | Demonstrate CKA readiness | Official blueprint, internal mocks, Killer.sh if included | Two unseen timed mocks, remediation closure, CKA attempt only after gate |
| 7 | Shift to application design and build | Start LFS259; Jobs, CronJobs, init/sidecar patterns, volumes, images | CKAD delta map, design lab, timed build task |
| 8 | Deploy, configure, and secure applications | LFS259; rollout, probes, resources, config, ServiceAccounts, security contexts | Deployment/config/security scenario set |
| 9 | Observe, network, and debug applications quickly | Finish LFS259; logs, probes, Services, policy, debugging | CKAD mixed debugging set and timed domain drills |
| 10 | Demonstrate CKAD readiness | Official blueprint, internal mocks, Killer.sh if included | Two unseen timed mocks, remediation closure, CKAD attempt only after gate |
| 11 | Repair the weakest result | Exam feedback categories and learner evidence | Closed remediation set or eligible retake readiness |
| 12 | Use contingency intentionally | Retake, schedule recovery, or publication hardening | Recorded outcomes, retrospective, bounded CKS/publication backlog |

## Weekly resource cap

Each week may adopt:

- one purchased-course spine;
- one curated official-document set;
- local labs/scenarios required by the gate;
- no more than two supplemental community resources.

Resource collection must not displace terminal practice.
